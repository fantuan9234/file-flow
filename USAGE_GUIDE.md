# File Flow 使用指南

## 📋 功能概述

File Flow 是一个基于 Electron + React + TypeScript 的批量文件重命名工具，提供直观的界面和强大的重命名规则配置。

## 🎨 界面布局

应用采用 Ant Design 的 Layout 组件，分为三个主要区域：

### 1. 左侧边栏 (Sider) - 400px

**文件夹选择区域**
- **拖拽上传**: 支持直接拖拽文件夹到上传区域
- **手动输入**: 可输入文件夹路径并按回车扫描
- **Scan 按钮**: 点击扫描指定文件夹

**文件列表区域**
- 显示扫描到的所有文件
- 支持复选框多选
- 显示文件路径、大小、修改时间
- 支持分页和排序
- 实时显示已选择的文件数量

### 2. 主内容区 (Content)

**规则配置卡片 (RuleConfig)**
- 重命名类型选择
- 参数输入
- 应用规则生成预览

**预览结果卡片**
- 显示重命名预览
- 执行重命名操作
- 撤销上次操作

## 🔧 重命名规则类型

### 1. 查找替换 (Find & Replace)

**功能**: 查找文件名中的指定文本并替换

**选项**:
- **Text 模式**: 普通文本查找替换
- **Regex 模式**: 正则表达式查找替换

**示例**:
```
查找：old
替换：new
结果：old_file.txt → new_file.txt
```

**正则示例**:
```
模式：(\d+)-(\d+)
替换：$2-$1
结果：2024-01-15 → 01-15-2024
标志：g (全局匹配)
```

### 2. 正则表达式 (Regular Expression)

**功能**: 使用正则表达式匹配并重命名整个文件名

**参数**:
- **Regex pattern**: 正则匹配模式
- **Replacement template**: 替换模板（支持捕获组 $1, $2 等）
- **Regex flags**: 正则标志（g, i, m 等）

**示例**:
```
模式：^(\w+)\s+(\w+)$
替换：$2_$1
结果：John Doe → Doe_John
标志：g
```

### 3. 添加前缀 (Add Prefix)

**功能**: 在文件名前添加指定前缀

**示例**:
```
前缀：backup_
结果：document.txt → backup_document.txt
```

### 4. 添加后缀 (Add Suffix)

**功能**: 在文件名后、扩展名前添加指定后缀

**示例**:
```
后缀：_v2
结果：document.txt → document_v2.txt
```

## 📖 使用流程

### 步骤 1: 选择文件夹

**方法一：拖拽**
1. 打开文件资源管理器
2. 选择目标文件夹
3. 拖拽到左侧的 "Click or drag folder to upload" 区域

**方法二：手动输入**
1. 在左侧输入框中输入文件夹路径
2. 按回车键或点击 "Scan" 按钮

### 步骤 2: 选择文件

1. 在文件列表中勾选需要重命名的文件
2. 支持多选（使用 Ctrl 或 Shift）
3. 查看底部显示已选择的文件数量

### 步骤 3: 配置规则

1. 选择重命名类型
2. 输入相应参数
3. 点击 "Apply Rules & Generate Preview"

### 步骤 4: 预览结果

1. 查看预览表格中的原路径和新路径
2. 确认重命名规则正确
3. 如有问题，返回步骤 3 调整规则

### 步骤 5: 执行操作

**执行重命名**:
- 点击 "Execute Rename" 按钮
- 等待执行完成
- 查看成功/失败统计

**撤销操作**:
- 点击 "Undo Last Operation" 按钮
- 文件将恢复到上次重命名前的状态

## 💡 使用技巧

### 1. 批量选择文件
- 使用复选框逐个选择
- 使用 Shift 点击进行范围选择
- 使用 Ctrl 点击进行多选

### 2. 正则表达式技巧
- `.` 匹配任意字符
- `\d` 匹配数字
- `\w` 匹配字母数字
- `^` 匹配开头
- `$` 匹配结尾
- `*` 零次或多次
- `+` 一次或多次
- `?` 零次或一次

### 3. 安全操作
- 先预览再执行
- 小批量测试后再大批量操作
- 重要文件先备份

### 4. 常见问题

**Q: 拖拽文件夹没反应？**
A: 确保拖拽的是文件夹而不是文件，或者使用手动输入路径方式。

**Q: 文件扫描不全？**
A: 某些目录（如 node_modules、.git 等）会被自动排除，这是正常行为。

**Q: 重命名失败？**
A: 检查目标路径是否已存在，或是否有权限写入。

**Q: 如何撤销操作？**
A: 点击 "Undo Last Operation" 按钮即可撤销上次的重命名操作。

## ⚠️ 注意事项

1. **权限问题**: 确保对目标文件夹有读写权限
2. **文件占用**: 确保要重命名的文件未被其他程序占用
3. **路径长度**: Windows 路径长度限制为 260 字符
4. **特殊字符**: 避免在文件名中使用特殊字符（\/:*?"<>|）
5. **重复名称**: 重命名后的文件不能与现有文件重名

## 🎯 快捷操作

- **Enter**: 在路径输入框中触发扫描
- **Ctrl + A**: 全选当前页文件（在表格中）
- **Esc**: 关闭消息提示

## 📊 状态说明

- **蓝色**: 可操作状态
- **灰色**: 禁用状态（无预览或无可撤销操作）
- **绿色**: 成功提示
- **红色**: 错误提示

## 🔄 文件刷新

执行重命名或撤销操作后，文件列表会自动刷新，显示最新状态。
